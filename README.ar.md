## ما هو Open Godot MCP؟

**Open Godot MCP** هو خادم [Model Context Protocol](https://modelcontextprotocol.io) مفتوح المصدر يربط مساعدات البرمجة بالذكاء الاصطناعي (Claude، GPT، Cursor، Windsurf، إلخ) بمحرر [Godot Engine](https://godotengine.org). يتيح للذكاء الاصطناعي **كتابة الكود، وتشغيل اللعبة، واختبار طريقة اللعب، والتصحيح عند نقاط التوقف، وفحص المتغيرات، والتحقق من الإصلاحات** — كل ذلك بشكل مستقل، دون تدخل بشري.

على عكس خوادم Godot MCP الحالية التي تعدل المشاهد فقط، يتيح Open Godot MCP للذكاء الاصطناعي **تشغيل اللعبة فعليًا** من خلال اختبار اللعب الحتمي (تجميد الساعة → خطوة زمنية → مراقبة الحالة → التحقق من النتيجة). وهو خادم Godot MCP **الوحيد** الذي يدعم **اختبار اللاعبين المتعددين**، و**تكامل مصحح DAP**، و**ذكاء كود LSP**.

| السمة | القيمة |
|-----------|-------|
| **نوع المشروع** | خادم MCP (Model Context Protocol) لمحرك Godot |
| **المحرك المستهدف** | Godot 4.5+ (دعم GDScript + C#) |
| **وقت التشغيل** | Python 3.11+ (الخادم) + GDScript (الإضافة) |
| **الرخصة** | MIT (100% مفتوح المصدر، لا نسخة مجانية محدودة) |
| **الأدوات** | ~30 أداة MCP، ~130 إجراء |
| **الميزات الرئيسية** | اختبار لعب حتمي، اختبار اللاعبين المتعددين، تصحيح DAP، LSP، كفاءة الرموز |
| **عملاء الذكاء الاصطناعي** | Claude Desktop، Cursor، Windsurf، VS Code (MCP)، Continue، Zed، أي عميل متوافق مع MCP |
| **المنصات** | Windows، macOS، Linux |
| **القدرات الفريدة** | اختبار اللاعبين المتعددين (لا يوجد خادم Godot MCP آخر يمتلك هذا)، تكامل DAP + LSP |

## لماذا وُجد هذا المشروع

كل خادم Godot MCP في السوق له نقائص:

| المشكلة | خوادم MCP الحالية | Open Godot MCP |
|---------|--------------|-----------------|
| الذكاء الاصطناعي لا يستطيع رؤية اللعبة وهي تعمل فعليًا | تعديل فقط، لا يمكن تشغيل اللعبة لإصلاح الأخطاء | **اختبار لعب حتمي** — تجميد الساعة، خطوة زمنية دقيقة، step_until حتى تحقق الشرط |
| اتصال غير مستقر | منفذ مثبت برمجيًا، لا نبضات، تعارضات WSL2 | منفذ قابل للضبط + نبضات + إعادة اتصال ذكية + تجنب تلقائي للمنفذ |
| لا يمكن اختبار اللاعبين المتعددين | جميع خوادم MCP تفتقر لاختبار اللاعبين المتعددين | **ميزة فريدة** — مثيلات متعددة، محاكاة أقران، التحقق من المزامنة، حقن ظروف الشبكة |
| إهدار الرموز | إرجاع كامل، PNG غير مضغوط، لا فروقات | مراقبة اقتصادية، ضغط لقطات الشاشة، فروقات، ملخصات، استعلامات تزايدية |
| إخصاء النسخة المجانية | النسخة المجانية محدودة، ادفع مقابل الميزات | **100% مفتوح المصدر برخصة MIT**، جميع الميزات مجانية |

## لمن هذا المشروع؟

- **مطورو الألعاب الذين يستخدمون Godot 4** الذين يريدون للذكاء الاصطناعي مساعدتهم في كتابة واختبار وتصحيح ألعابهم
- **المبرمجون بمساعدة الذكاء الاصطناعي** (مستخدمو Claude، Cursor، Windsurf، VS Code MCP) العاملون على مشاريع Godot
- **استوديوهات الألعاب المستقلة** التي تحتاج لاختبار لعب آلي دون كتابة أطر اختبار
- **مطورو ألعاب اللاعبين المتعددين** الذين يحتاجون لاختبار مزامنة الشبكة، وزمن الوصول، وسلوك الأقران
- **مدافعو المصدر المفتوح** الذين يريدون خادم MCP مجاني بالكامل دون جدار دفع

## حالات الاستخدام

| حالة الاستخدام | كيف يساعد Open Godot MCP |
|----------|--------------------------|
| **الذكاء الاصطناعي يصلح خطأ في الحركة** | الذكاء الاصطناعي يضع نقطة توقف → يشغل اللعبة → يفحص المتغيرات → يحدد السبب الجذري → يصلح الكود → يعيد الاختبار |
| **اختبار آلي لمعركة الزعيم** | تجميد الساعة → استدعاء الزعيم → خطوة زمنية → محاكاة إدخال التفادي → التحقق من بقاء اللاعب |
| **التحقق من مزامنة اللاعبين المتعددين** | تشغيل مثيلات المضيف + العميل → حقن زمن الوصول → مقارنة حالة المزامنة → كشف أخطاء فقدان المزامنة |
| **تحليل الأداء** | التقاط لقطة مُحلل الأداء → تحديد القمة → التحسين → إعادة القياس |
| **اختبار الانحدار** | تشغيل مجموعة الاختبار بعد تغيير الكود → التأكد من تطابق حالة اللعبة مع المتوقع |
| **تكرار تصميم المراحل** | الذكاء الاصطناعي ينشئ العقد → يرتب المشهد → يشغل اللعبة → يلتقط لقطة شاشة → يعدّل |

## القدرات الأساسية

### 1. اختبار اللعب الحتمي (يحل مشكلة "الذكاء الاصطناعي لا يستطيع رؤية اللعبة وهي تعمل")

الذكاء الاصطناعي لا يكتب الكود فقط — بل يمكنه **تشغيل اللعبة بنفسه للتحقق من الإصلاحات**:

```
godot_game play frozen=true                    # تشغيل اللعبة (ساعة مجمّدة)
godot_exec eval code="GameState.wave = 3"      # إعداد سيناريو الاختبار
godot_game_time step_until "boss.size() >= 1"  # انتظار ظهور الزعيم
godot_runtime_state digest                     # مراقبة الحالة (JSON، بدون رموز رؤية)
godot_game_time step ms=500 + dodge input      # لعب اللحظة الحرجة
godot_screenshot game                          # لقطة شاشة فقط عندما تستحق
```

### 2. اختبار اللاعبين المتعددين (ميزة فريدة — لا يوجد خادم Godot MCP آخر يمتلكها)

قدرة لا يمتلكها أي خادم Godot MCP حالي:

```
godot_network launch_instance role="host"      # بدء الخادم
godot_network launch_instance role="client"    # بدء العميل
godot_network network_condition latency=200    # حقن تأخير 200 مللي ثانية
godot_network sync_state                       # التحقق من مزامنة المثيلات المتعددة
godot_network simulate_peer count=50           # اختبار ضغط بـ 50 قرينًا
```

### 3. كفاءة الرموز

كل أداة مصممة لتوفير الرموز:

- **مراقبة اقتصادية**: ملخص حالة JSON يحل محل لقطات الشاشة (يوفر 90% من الرموز)
- **إرجاع الفروقات**: إرجاع الأجزاء المتغيرة فقط
- **ضغط لقطات الشاشة**: JPEG/WebP + حفظ على القرص (ليس في السياق)
- **فصل القراءة/الكتابة**: القراءة مسموحة تلقائيًا، الكتابة محمية
- **عمليات دفعية**: إتمام عمليات متعددة في رحلة واحدة ذهابًا وإيابًا

### 4. استقرار الاتصال

يحل مشكلة "لا يمكن الاتصال" في خوادم MCP الحالية:

- منفذ قابل للضبط (متغيرات البيئة > إعدادات المحرر > التجنب التلقائي)
- كشف حجز المنفذ في ويندوز (تجنب المنافذ المحجوزة لـ Hyper-V/WSL2/Docker)
- آلية النبضات (كشف استباقي للاتصالات الميتة)
- إعادة اتصال ذكية (تراجع أسي + حد أقصى للمحاولات + إشعار في الواجهة)

### 5. تصحيح كامل

- **DAP (بروتوكول محول المصحح)**: نقاط التوقف، التنقل خطوة بخطوة، فحص المتغيرات (stack_trace، variables، evaluate)
- **LSP (بروتوكول خادم اللغة)**: تشخيصات ثابتة، إكمال تلقائي، الانتقال إلى التعريف
- **مُحلل الأداء**: لقطات أداء، تحليل الجدول الزمني، كشف القمم

## الأسئلة الشائعة

### ما هو Model Context Protocol (MCP)؟

[Model Context Protocol](https://modelcontextprotocol.io) هو معيار مفتوح يتيح لمساعدات الذكاء الاصطناعي الاتصال بأدوات ومصادر بيانات خارجية. Open Godot MCP هو خادم MCP يربط الذكاء الاصطناعي بمحرر Godot Engine.

### ما إصدارات Godot المدعومة؟

Godot 4.5 وأحدث. تستخدم الإضافة واجهات Godot 4.x البرمجية بما في ذلك `EditorDebuggerPlugin` و`EditorInspector` وقناة رسائل المصحح.

### ما عملاء الذكاء الاصطناعي المتوافقون؟

أي عميل متوافق مع MCP: Claude Desktop، Cursor، Windsurf، VS Code (مع إضافة MCP)، Continue، Zed، وأي عميل يدعم معيار Model Context Protocol.

### هل يدعم C# (إصدار Godot .NET)؟

نعم. فحص صياغة C# والتحقق من التجميع مدعومان. راجع [Docs/08-CSharp-Support/](Docs/08-CSharp-Support/).

### كيف يختلف هذا عن خوادم Godot MCP الأخرى؟

Open Godot MCP هو خادم Godot MCP **الوحيد** الذي يدعم اختبار اللاعبين المتعددين، وتكامل مصحح DAP (نقاط التوقف، تتبع المكدس، فحص المتغيرات)، وذكاء كود LSP. كما أنه يمتلك التصميم الأشمل لكفاءة الرموز.

### هل هو مجاني فعلًا؟

نعم. 100% برخصة MIT، لا نموذج نسخة مجانية محدودة، لا جدار دفع، لا قيود على الميزات. جميع الميزات مجانية للجميع.

### هل يمكن للذكاء الاصطناعي تشغيل اللعبة فعليًا؟

نعم. من خلال اختبار اللعب الحتمي، يمكن للذكاء الاصطناعي تجميد ساعة اللعبة، وتقديم الوقت بخطوات دقيقة، وحقن سيناريوهات الاختبار، ومحاكاة إدخال اللاعب، ومراقبة حالة اللعبة كـ JSON، والتقاط لقطات شاشة — كل ذلك للتحقق من أن تغييرات الكود تعمل بشكل صحيح.

### كيف يعمل اختبار اللاعبين المتعددين؟

يمكن لـ Open Godot MCP تشغيل عدة مثيلات Godot (مضيف + عملاء)، ومحاكاة الأقران، وحقن ظروف الشبكة (زمن الوصول، فقدان الحزم)، والتحقق من أن حالة اللعبة متزامنة عبر المثيلات.

## شكر وتقدير

يقف Open Godot MCP على أكتاف العمالقة، آخذًا أفضل ما لديهم من:

- [Coding-Solo/godot-mcp](https://github.com/Coding-Solo/godot-mcp) (4.8k نجمة) — البنية الأساسية
- [satelliteoflove/godot-mcp](https://github.com/satelliteoflove/godot-mcp) — اختبار اللعب الحتمي، المراقبة الاقتصادية، فصل القراءة/الكتابة
- [hi-godot/godot-ai](https://github.com/hi-godot/godot-ai) — قناة المصحح في وقت التشغيل، التراجع/الإعادة، حجز منفذ ويندوز، 20+ إعداد عميل، McpTestSuite
- [tomyud1/godot-mcp](https://github.com/tomyud1/godot-mcp) — بنية القناة المزدوجة، تسلسل Variant، حماية الحذف
- [rosskarchner/godot-mcp](https://github.com/rosskarchner/godot-mcp) — تكامل DAP + LSP، المثيلات المتعددة، عزل المنفذ
- [Erodenn/godot-mcp-runtime](https://github.com/Erodenn/godot-mcp-runtime) — بصمة صفرية، مفهوم Playwright لـ Godot
- [thediymaker/godot-mcp](https://github.com/thediymaker/godot-mcp) — مرجع اتساع 149 أداة
