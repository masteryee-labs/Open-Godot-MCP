## Что такое Open Godot MCP?

**Open Godot MCP** — это открытый сервер [Model Context Protocol](https://modelcontextprotocol.io), который подключает ИИ-ассистентов для кода (Claude, GPT, Cursor, Windsurf и др.) к редактору [Godot Engine](https://godotengine.org). Он позволяет ИИ **писать код, запускать игру, тестировать геймплей, отлаживать по точкам останова, инспектировать переменные и проверять исправления** — всё автономно, без вмешательства человека.

В отличие от существующих Godot MCP-серверов, которые только редактируют сцены, Open Godot MCP позволяет ИИ **фактически играть в игру** через детерминированное плейтестирование (заморозка часов → пошаговое продвижение времени → наблюдение состояния → проверка результата). Это **единственный** Godot MCP, поддерживающий **тестирование мультиплеера**, **интеграцию отладчика DAP** и **интеллект кода LSP**.

| **ИИ-клиенты** | Claude Desktop, Cursor, Windsurf, VS Code (MCP), Continue, Zed, любой MCP-совместимый клиент |
| **Платформы** | Windows, macOS, Linux |
| **Уникальные возможности** | Тестирование мультиплеера (нет ни у одного другого Godot MCP), интеграция DAP + LSP |

## Зачем это существует

У каждого Godot MCP на рынке есть недостатки:

| Проблема | Существующие MCP | Open Godot MCP |
|---------|--------------|-----------------|
| ИИ не видит, как игра реально запускается | Только редактирование, нельзя запустить игру для исправления багов | **Детерминированное плейтестирование** — заморозка часов, пошаговое продвижение времени, step_until по условию |
| Нестабильное соединение | Жёстко заданный порт, без пульса, конфликты с WSL2 | Настраиваемый порт + пульс + умное переподключение + автоизбегание портов |
| Нет тестирования мультиплеера | Все MCP лишены тестирования мультиплеера | **Уникально** — многоэкземплярность, симуляция пиров, проверка синхронизации, инъекция сетевых условий |
| Расход токенов | Полные возвраты, несжатый PNG, без диффов | Дешёвое наблюдение, сжатие скриншотов, диффы, сводки, инкрементальные запросы |
| Урезанный freemium | Бесплатная версия ограничена, платите за функции | **100% MIT open source**, все функции бесплатно |

## Ключевые возможности

### 1. Детерминированное плейтестирование (решает «ИИ не видит запущенную игру»)

ИИ не просто пишет код — он может **сам играть в игру, чтобы проверить исправления**:

```
godot_game play frozen=true                    # Запустить игру (замороженные часы)
godot_exec eval code="GameState.wave = 3"      # Настроить тестовый сценарий
godot_game_time step_until "boss.size() >= 1"  # Дождаться появления босса
godot_runtime_state digest                     # Наблюдать состояние (JSON, без vision-токенов)
godot_game_time step ms=500 + dodge input      # Отыграть ключевой момент
godot_screenshot game                          # Скриншот только когда это имеет смысл
```

### 2. Тестирование мультиплеера (уникальная возможность — нет ни у одного другого Godot MCP)

Возможность, которой нет ни у одного существующего Godot MCP:

```
godot_network launch_instance role="host"      # Запустить сервер
godot_network launch_instance role="client"    # Запустить клиент
godot_network network_condition latency=200    # Внести задержку 200 мс
godot_network sync_state                       # Проверить синхронизацию экземпляров
godot_network simulate_peer count=50           # Стресс-тест 50 пиров
```

### 3. Экономия токенов

Каждый инструмент спроектирован для экономии токенов:

- **Дешёвое наблюдение**: JSON-дайджест состояния заменяет скриншоты (экономия 90% токенов)
- **Дифф-возвраты**: Возвращаются только изменённые части
- **Сжатие скриншотов**: JPEG/WebP + сохранение на диск (не в контексте)
- **Разделение чтения/записи**: чтение авто-разрешено, запись под контролем
- **Пакетные операции**: Выполнение нескольких операций за один round-trip

### 4. Стабильность соединения

Решает проблему «не удаётся подключиться» в существующих MCP:

- Настраиваемый порт (env > EditorSettings > автоизбегание)
- Обнаружение резервирования портов Windows (избегать зарезервированных портов Hyper-V/WSL2/Docker)
- Механизм пульса (проактивное обнаружение мёртвых соединений)
- Умное переподключение (экспоненциальная задержка + макс. попыток + уведомление в UI)

### 5. Полноценная отладка

- **DAP (Debugger Adapter Protocol)**: точки останова, пошаговое выполнение, инспекция переменных (stack_trace, variables, evaluate)
- **LSP (Language Server Protocol)**: статическая диагностика, автодополнение, переход к определению
- **Профайлер**: снимки производительности, анализ таймлайна, обнаружение всплесков

## Быстрый старт

### 1. Установите MCP-сервер

```bash
uv tool install open-godot-mcp
# или
pip install open-godot-mcp
```

### 2. Настройте AI-клиент

```json
{
  "mcpServers": {
    "open-godot-mcp": {
      "command": "open-godot-mcp"
    }
  }
}
```

### 3. Откройте проект Godot

Аддон внедряется автоматически. Откройте ваш AI-клиент и начните использовать.

## FAQ

### Что такое Model Context Protocol (MCP)?

[Model Context Protocol](https://modelcontextprotocol.io) — это открытый стандарт, позволяющий ИИ-ассистентам подключаться к внешним инструментам и источникам данных. Open Godot MCP — это MCP-сервер, который подключает ИИ к редактору Godot Engine.

### Какие версии Godot поддерживаются?

Godot 4.5 и новее. Аддон использует API Godot 4.x, включая `EditorDebuggerPlugin`, `EditorInspector` и канал сообщений отладчика.

### Какие ИИ-клиенты совместимы?

Любой MCP-совместимый клиент: Claude Desktop, Cursor, Windsurf, VS Code (с расширением MCP), Continue, Zed и любой клиент, поддерживающий стандарт Model Context Protocol.

### Поддерживается ли C# (.NET-версия Godot)?

Да. Поддерживаются проверка синтаксиса C# и верификация компиляции. См. [Docs/08-CSharp-Support/](Docs/08-CSharp-Support/).

### Чем это отличается от других Godot MCP-серверов?

Open Godot MCP — **единственный** Godot MCP, поддерживающий тестирование мультиплеерных игр, интеграцию отладчика DAP (точки останова, стек вызовов, инспекция переменных) и интеллект кода LSP. Также он имеет самую всестороннюю систему экономии токенов.

### Это действительно бесплатно?

Да. 100% MIT-лицензия, без freemium-модели, без платного контента, без ограничения функций. Все функции бесплатны для всех.

### Может ли ИИ действительно играть в игру?

Да. Через детерминированное плейтестирование ИИ может заморозить игровые часы, пошагово продвигать время, внедрять тестовые сценарии, симулировать ввод игрока, наблюдать состояние игры в виде JSON и делать скриншоты — всё для проверки того, что изменения кода работают корректно.

### Как работает тестирование мультиплеера?

Open Godot MCP может запустить несколько экземпляров Godot (хост + клиенты), симулировать пиров, внедрять сетевые условия (задержка, потеря пакетов) и проверять, что состояние игры синхронизировано между экземплярами.
