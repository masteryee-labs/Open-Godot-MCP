## Що таке Open Godot MCP?

**Open Godot MCP** — сервер [Model Context Protocol](https://modelcontextprotocol.io) з відкритим кодом, що з'єднує ШІ-помічники кодування (Claude, GPT, Cursor, Windsurf тощо) з редактором [Godot Engine](https://godotengine.org). Він дозволяє ШІ **писати код, запускати гру, тестувати геймплей, дебажити на брейкпоінтах, інспектувати змінні та перевіряти виправлення** — все автономно, без втручання людини.

На відміну від існуючих Godot MCP-серверів, які лише редагують сцени, Open Godot MCP дозволяє ШІ **фактично грати в гру** через детермінований плейтестинг (заморожування годинника → покрокове просування часу → спостереження стану → перевірка результату). Це **єдиний** Godot MCP, що підтримує **тестування мультиплеєра**, **інтеграцію DAP-дебаггера** та **код-інтелект LSP**.

| Атрибут | Значення |
|-----------|-------|
| **Тип проєкту** | MCP-сервер (Model Context Protocol) для Godot Engine |
| **Цільовий рушій** | Godot 4.5+ (підтримка GDScript + C#) |
| **Runtime** | Python 3.11+ (сервер) + GDScript (аддон) |
| **Ліцензія** | MIT (100% відкритий код, без freemium) |
| **Інструменти** | ~30 інструментів MCP, ~130 дій |
| **Ключові можливості** | Детермінований плейтестинг, тестування мультиплеєра, DAP-дебаггінг, LSP, ефективність токенів |
| **ШІ-клієнти** | Claude Desktop, Cursor, Windsurf, VS Code (MCP), Continue, Zed, будь-який MCP-сумісний клієнт |
| **Платформи** | Windows, macOS, Linux |
| **Унікальні можливості** | Тестування мультиплеєра (немає в жодному іншому Godot MCP), інтеграція DAP + LSP |

## Чому це існує

Кожен Godot MCP на ринку має недоліки:

| Проблема | Існуючі MCP | Open Godot MCP |
|---------|--------------|-----------------|
| ШІ не бачить, як гра насправді запускається | Тільки редагування, не може запустити гру для виправлення багів | **Детермінований плейтестинг** — заморожування годинника, покрокове просування часу, step_until за умовою |
| Нестабільне з'єднання | Жорстко заданий порт, без heartbeat, конфлікти WSL2 | Налаштовуваний порт + heartbeat + розумне перепідключення + автоунікання портів |
| Неможливо тестувати мультиплеєр | Усі MCP не мають тестування мультиплеєра | **Унікально** — мультиінстанс, симуляція пірів, перевірка синхронізації, ін'єкція мережевих умов |
| Марнування токенів | Повні повернення, нестиснені PNG, без дифів | Дешеве спостереження, стиснення скріншотів, дифи, підсумки, інкрементальні запити |
| Урізаний безкоштовний функціонал | Безкоштовна версія обмежена, плати за функції | **100% MIT відкритий код**, усі функції безкоштовні |

## Для кого це?

- **Розробники ігор на Godot 4**, які хочуть, щоб ШІ допомагав писати, тестувати та дебажити їхні ігри

- **Розробники мультиплеєрних ігор**, яким потрібно тестувати мережеву синхронізацію, затримку та поведінку пірів
- **Прихильники відкритого коду**, які хочуть повністю безкоштовний MCP-сервер без paywall

## Сценарії використання

| Сценарій | Як Open Godot MCP допомагає |
|----------|--------------------------|
| **ШІ виправляє баг руху** | ШІ ставить брейкпоінт → запускає гру → інспектує змінні → знаходить причину → виправляє код → повторно тестує |
| **Автоматизоване тестування бос-файту** | Заморозити годинник → спавн боса → покроковий час → симуляція вводу ухилення → перевірка виживання гравця |
| **Перевірка синхронізації мультиплеєра** | Запуск host + клієнт-інстансів → ін'єкція затримки → порівняння стану синхронізації → виявлення десинхрон-багів |
| **Профілювання продуктивності** | Знімок профайлера → виявлення спайку → оптимізація → повторний замір |
| **Регресійне тестування** | Запуск тестового набору після зміни коду → перевірка відповідності стану гри очікуваному |
| **Ітерація дизайну рівнів** | ШІ створює вузли → розставляє сцену → запускає гру → скріншот результату → коригує |

## Основні можливості

### 1. Детермінований плейтестинг (вирішує «ШІ не бачить запущену гру»)

ШІ не лише пише код — він може **самостійно грати в гру, щоб перевірити виправлення**:

```
godot_game play frozen=true                    # Запустити гру (заморожений годинник)
godot_exec eval code="GameState.wave = 3"      # Налаштувати тестовий сценарій
godot_game_time step_until "boss.size() >= 1"  # Дочекатися появи боса
godot_runtime_state digest                     # Спостерігати стан (JSON, без візуальних токенів)
godot_game_time step ms=500 + dodge input      # Зіграти ключовий момент
godot_screenshot game                          # Скріншот лише коли це варто
```

### 2. Тестування мультиплеєра (унікальна функція — немає в жодному іншому Godot MCP)

Можливість, якої немає в жодному існуючому Godot MCP:

```
godot_network launch_instance role="host"      # Запустити сервер
godot_network launch_instance role="client"    # Запустити клієнт
godot_network network_condition latency=200    # Ін'єктувати затримку 200мс
godot_network sync_state                       # Перевірити синхронізацію мультиінстансу
godot_network simulate_peer count=50           # Стрес-тест 50 пірів
```

### 3. Ефективність токенів

Кожен інструмент має дизайн для економії токенів:

- **Дешеве спостереження**: JSON-дайджест стану замість скріншотів (економить 90% токенів)
- **Повернення дифів**: Повертаються лише змінені частини
- **Стиснення скріншотів**: JPEG/WebP + збереження на диск (не в контексті)
- **Розділення читання/запису**: читання авто-дозволене, запис обмежений
- **Пакетні операції**: Виконати кілька операцій за один round-trip

### 4. Стабільність з'єднання

Вирішує проблему «не можу підключитися» в існуючих MCP:

- Налаштовуваний порт (env > EditorSettings > автоунікання)
- Виявлення резервування портів Windows (уникнення зарезервованих портів Hyper-V/WSL2/Docker)
- Механізм heartbeat (проактивне виявлення мертвих з'єднань)
- Розумне перепідключення (експоненційний backoff + макс. спроб + UI-сповіщення)

### 5. Повний дебаггінг

- **DAP (Debugger Adapter Protocol)**: брейкпоінти, покрокове виконання, інспекція змінних (stack_trace, variables, evaluate)
- **LSP (Language Server Protocol)**: статична діагностика, автодоповнення, перехід до визначення
- **Профайлер**: знімки продуктивності, аналіз таймлайну, виявлення спайків

## FAQ

### Що таке Model Context Protocol (MCP)?

[Model Context Protocol](https://modelcontextprotocol.io) — відкритий стандарт, що дозволяє ШІ-асистентам підключатися до зовнішніх інструментів та джерел даних. Open Godot MCP — MCP-сервер, що з'єднує ШІ з редактором Godot Engine.

### Які версії Godot підтримуються?

Godot 4.5 та новіші. Аддон використовує API Godot 4.x, включно з `EditorDebuggerPlugin`, `EditorInspector` та каналом повідомлень дебаггера.

### Які ШІ-клієнти сумісні?

Будь-який MCP-сумісний клієнт: Claude Desktop, Cursor, Windsurf, VS Code (з MCP-розширенням), Continue, Zed та будь-який клієнт, що підтримує стандарт Model Context Protocol.

### Чи підтримується C# (.NET-версія Godot)?

Так. Підтримуються перевірка синтаксису C# та перевірка компіляції. Див. [Docs/08-CSharp-Support/](Docs/08-CSharp-Support/).

### Чим це відрізняється від інших Godot MCP-серверів?

Open Godot MCP — **єдиний** Godot MCP, що підтримує тестування мультиплеєрних ігор, інтеграцію DAP-дебаггера (брейкпоінти, стек-трейси, інспекція змінних) та код-інтелект LSP. Також він має найбільш всебічний дизайн ефективності токенів.

### Це дійсно безкоштовно?

Так. 100% MIT-ліцензія, без freemium-моделі, без paywall, без обмеження функцій. Усі функції безкоштовні для всіх.

### Чи може ШІ фактично грати в гру?

Так. Через детермінований плейтестинг ШІ може заморозити ігровий годинник, покроково просувати час, ін'єктувати тестові сценарії, симулювати ввод гравця, спостерігати стан гри як JSON та робити скріншоти — все для перевірки, що зміни коду працюють коректно.

### Як працює тестування мультиплеєра?

Open Godot MCP може запустити кілька інстансів Godot (host + клієнти), симулювати пірів, ін'єктувати мережеві умови (затримка, втрата пакетів) та перевіряти, що стан гри синхронізований між інстансами.

## Подяки

Open Godot MCP стоїть на плечах гігантів, беручи найкраще з:

- [Coding-Solo/godot-mcp](https://github.com/Coding-Solo/godot-mcp) (4.8k зірок) — фундаментальна архітектура
- [satelliteoflove/godot-mcp](https://github.com/satelliteoflove/godot-mcp) — детермінований плейтестинг, дешеве спостереження, розділення читання/запису
- [hi-godot/godot-ai](https://github.com/hi-godot/godot-ai) — runtime через канал дебаггера, Undo/Redo, резервування портів Windows, 20+ конфіг клієнтів, McpTestSuite
- [tomyud1/godot-mcp](https://github.com/tomyud1/godot-mcp) — двоканальна архітектура, серіалізація Variant, захист від видалення
- [rosskarchner/godot-mcp](https://github.com/rosskarchner/godot-mcp) — інтеграція DAP + LSP, мультиінстанс, ізоляція портів
- [Erodenn/godot-mcp-runtime](https://github.com/Erodenn/godot-mcp-runtime) — нульовий слід, концепція Playwright для Godot
- [thediymaker/godot-mcp](https://github.com/thediymaker/godot-mcp) — еталон широти 149 інструментів
